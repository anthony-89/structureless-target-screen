A structureless heart-failure target — finding where its activator binds, without pre-deciding the answer.

Heart failure represses OPLAH (5-oxoprolinase, UniProt O14841, 1288 aa); a screen of FDA-approved compounds found 5′-AMP activates it — but it has no solved structure and no known binding site. With Claude Code we find the site by unbiased search, then screen it for mimics and better binders. A reusable pipeline for any structureless target with one known modulator.

✓ Q1 structure ✓ Q5 pockets ✓ Q2 AMP site ✓ Q3 mimics ✓ Q4 better binders
Why this target

Heart failure switches OPLAH off → the toxic metabolite 5-oxoproline accumulates → oxidative stress damages the heart (a murine model of HFpEF). So activating OPLAH is a therapeutic goal — and 5′-AMP, which we identified in our prior work from a screen of 1280 FDA-approved compounds, is the activator to build on. But with no structure and no known site, you can't design a better one. That's where this pipeline starts.

✗ The bias we removed

Copy the binding site from a homolog → it pre-decides the answer: it can only confirm the guess, never discover AMP binds elsewhere. (Blind-docking with DiffDock caps input at 1022 residues, forcing a truncation that deletes residues forming the real pocket.)

✓ What we did instead

Detect every pocket over the full structure by geometry, then let AMP dock freely and choose its own site. The answer is now an output of unbiased search, not an input.

One pocket, built from the whole 1288-aa chain ▬ pocket residues   ┊ DiffDock 1022 cap
The pocket draws its walls from residues 13 → 1288. DiffDock's 1022 cap would delete the C-terminal wall — truncation isn't just risky, it physically breaks this site.
Background: OPLAH is repressed in heart failure and 5-oxoproline drives cardiac oxidative stress (van der Pol et al., Sci Transl Med 2017; Cardiovasc Res 2018). In our own prior work, 5′-AMP enhances OPLAH ATPase >50% in a screen of 1280 FDA-approved compounds (Esquivel-Gaytan et al., Advanced Therapeutics 2026).

→ Beyond OPLAH

The method assumes nothing about the target, so it generalizes: hand it any protein with no structure and one known modulator, and it returns a testable shortlist — the pattern behind most hard drug targets. OPLAH is the first case; the pocket-detection step folds back into the pipeline (module M4), driven end-to-end by Claude Code.
